# Backup & Zugriffskontrolle

## Website (Vercel / Git)

| Aspekt | Maßnahme |
|--------|----------|
| Quellcode | Git-Repository (GitHub), Versionshistorie |
| Deployment | Vercel – automatische Builds bei Push |
| Backup | Git-Remote als primäres Backup; Vercel Deployment-History |
| Zugriff | Nur autorisierte Entwickler mit Git/Vercel-Zugang |
| 2FA | Auf GitHub- und Vercel-Konten aktivieren |

## E-Mail-Postfächer

| Aspekt | Maßnahme |
|--------|----------|
| Zugriff | Nur Ordinationsteam mit individuellen Zugängen |
| Passwörter | Stark, einzigartig, 2FA wo möglich |
| Backup | Provider-Backup prüfen (IMAP/Archiv) |

## Ordinations-IT (allgemein)

- Keine Patientendaten auf der öffentlichen Website
- Ärztliche Dokumentation in separatem Praxis-System (nicht Teil dieser Website)

## Prüfintervall
Halbjährlich: Wer hat Zugriff auf Vercel, Git, E-Mail?
